### 浏览器截图预览修复
- **问题**：远程服务器上最终回复中截图引用绝对路径，导致图片无法加载
- **根因**：工具输出的 `/api/screenshot?path=%2Froot%2F...` 中 URL 编码路径被 LLM 解码还原，LLM 在最终回复中直接引用绝对路径
- **修复**：改为文件名 token 方案——token = `screenshot_123`（文件名不包含扩展名），端点通过 `workspace/.browser_screenshots/{token}.png` 直接查找，零状态、零依赖、跨进程可访问
- **影响文件**：`agent_core/tools/browser_tools.py`、`agent_core/main.py`
- **补充修复**：`/api/screenshot` 端点使用 `config.workspace`（默认工作区），但工具使用 `_workspace_for_user(uid)`（用户工作区，含 uid 子目录），路径不一致导致文件找不到。改用 `_workspace_for_user(uid)` 对齐。

### 新增拖拽/滑块工具
- `browser_drag(source, target)`: 基于 Playwright drag_to 的标准拖拽
- `browser_slide(selector, offset_x, offset_y)`: 模拟人类滑动轨迹（贝塞尔缓动 + 随机抖动），用于滑块验证码
