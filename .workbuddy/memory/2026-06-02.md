# 2026-06-02 工作日志

## 日志机制接入
- 创建 `agent_core/logger.py`：统一日志模块，基于 `TimedRotatingFileHandler` 实现每 7 天自动滚动轮换，保留 4 个备份
- 日志路径：`~/.desktop_agent/logs/agent.log`，格式 `时间 | 级别 | 模块名 | 消息`
- 日志级别可通过环境变量 `AGENT_LOG_LEVEL` 配置（默认 INFO）
- 替换 `main.py` 和 `agent.py` 中所有 `print()` 为 `logger.info()` / `logger.error()`
- 关键异常点增加 `logger.exception()` 记录堆栈
- Uvicorn 日志集成到统一的日志配置中
- 更新 `start.sh` 显示结构化日志路径和滚动说明

## 工具调用失败展示优化
- 修复 agent.py 中 `tool_result` SSE 事件缺失 `step` 字段，前端无法定位到对应工具卡片的 bug
- 新增 `error` 标记：检测工具返回结果是否以 `❌` 开头，自动标记为失败状态
- 前端 tool_result 处理：失败时显示红色「❌ 失败」状态，错误信息展示在卡片详情内（红底），不再前置 `✅`
- 新增 CSS：`.step-status.error`（红色文字）、`.result-error`（红底白字），与成功状态的绿底绿字区分
- 新增中英文翻译 key：`failed: '❌ 失败' / '❌ Failed'`

## 修复并行工具步进跟踪（第二轮）
- **根因**：`active_step` 是单变量，LangGraph 中多工具并行执行时，后启动的工具会覆盖前者的 step 值；且每个 `on_tool_end` 后 `active_step` 被重置为 0，导致后续并行工具的 `tool_result` step 全为 0，前端找不到卡片，工具永久卡在"执行中"
- **修复**：改用 `running_tools: dict[str, dict]`（keyed by LangGraph 事件 `run_id`），每个工具独立记录 `{name, step, started_at}`
  - `on_tool_start`：添加到字典，不再用单变量
  - `on_tool_end`：通过 `run_id` 从字典精确查找该工具的 step，`pop` 移除
  - `progress` 事件：遍历 `running_tools` 中所有正在运行的工具，为每个工具独立发送进度
