# AgentSmith

[ä¸­ć–‡](README.md) | [English](README.en.md)

AgentSmith is a locally hosted, private desktop AI agent. It is built with FastAPI, LangGraph, and OpenAI-compatible model APIs, and provides chat-based task execution, tool calls, downloadable artifacts, Skills extensions, multi-user isolation, long-term memory, and visible execution steps.

It works well as a personal or intranet team assistant for reading and writing workspace files, running Python, searching the web, managing Git, generating documents, analyzing images, and delegating independent tasks to subagents.

---

## Core Features

| Feature | Description |
| --- | --- |
| Chat agent | LangGraph ReAct Agent with streaming output, reasoning/tool step display, long-task progress hints, and task cancellation |
| Multiline and image input | The composer supports multiline text and pasted images; images are used only for the current turn and are not written to history as base64 |
| Multimodal model switching | For MiMo models, image turns can temporarily switch to `mimo-v2.5` when the current model does not support images |
| File tools | Read, write, append, delete, list, and search workspace files; large files return summaries and paths instead of flooding context |
| Artifacts | Files generated by the AI automatically get download links; Markdown files can be previewed in a modal |
| Python execution | Run Python code and return output; very large output is summarized with head and tail excerpts |
| Web tools | `web_search` for search and `web_fetch` for page text, with retries and fallback requests |
| Git tools | Inspect status, diff, log, and show; add, commit, push, and revert only on explicit user request |
| Subagents | `delegate_task` can delegate to coder, reviewer, and debugger agents; the current MVP runs synchronously |
| Skills | Loads `SKILL.md`, compatible with YAML frontmatter and common oh-my-openagent / Superpowers-style skills |
| Long-term memory | Stores user preferences, project facts, and reusable environment notes per user |
| Multi-user | Login protection, admin user management, and isolated workspaces, sessions, usage records, and memories per user |
| Context management | Estimates model context usage and compresses history near the threshold; large logs/files are not inserted in full |
| Usage tracking | Tracks calls and tokens by user, session, provider, model, and tool |

## Quick Start

### Requirements

- Python 3.9+ (3.10+ recommended; 3.9 uses `from __future__ import annotations` for PEP 604 syntax)
- An OpenAI-compatible model API key
- Optional: Git, curl, and a Windows packaging environment

### Start

```bash
git clone https://gitee.com/mingwuda/desktop-agent.git
cd desktop-agent
pip install -r requirements.txt
./start.sh
```

Open the app after startup:

```text
http://127.0.0.1:8899/
```

You can also start the backend directly:

```bash
cd agent_core
python main.py
```

On Windows:

```cmd
start.cmd
```

The first startup creates configuration, authentication, and user data directories. If no user is configured, it creates `admin / admin123`; change this immediately for real deployments.

## Login and Multi-User

Login protection is enabled by default. The auth file is stored at:

```text
~/.desktop_agent/auth.json
```

You can initialize login settings through environment variables:

```bash
export DESKTOP_AGENT_AUTH_USER=admin
export DESKTOP_AGENT_AUTH_PASSWORD='your-strong-password'
export DESKTOP_AGENT_AUTH_SECRET='replace-with-a-random-secret'
```

Generate a short-lived passwordless login link:

```bash
python generate_login_url.py --host 127.0.0.1 --port 8899 --expires 300 --user admin
```

Optional flags:

- `--qr`: show a QR code in the terminal
- `--copy`: copy the link to the clipboard

User data is isolated per user:

```text
~/.desktop_agent/users/{user_id}/sessions/sessions.sqlite3
~/.desktop_agent/users/{user_id}/usage/usage.sqlite3
~/.desktop_agent/users/{user_id}/memory/
~/agent_workspace/{user_id}/
```

Only the `admin` user can access settings and user-management entries.

## Model Configuration

Open **Settings** in the top-right corner to configure:

| Field | Description |
| --- | --- |
| Model provider | OpenAI, DeepSeek, Qwen, or a custom provider |
| API Key | Model service key |
| Model name | Examples: `gpt-4o`, `deepseek-chat`, `qwen-plus`, `mimo-v2.5-pro` |
| API base URL | OpenAI-compatible endpoint; leave blank to use OpenAI defaults |
| Max reasoning steps | LangGraph recursion limit for one task, default `60` |
| Request retries | Model connection retry count, default `3` |
| Request timeout | Model read timeout, default `30` seconds |
| Context window | Max context length for the current model; inferred from model name when blank |

Configuration is saved to:

```text
~/.desktop_agent/config.json
```

Common environment variables:

| Variable | Description | Default |
| --- | --- | --- |
| `LLM_PROVIDER` | Current provider | `openai` |
| `LLM_API_KEY` / `OPENAI_API_KEY` | API key | empty |
| `LLM_MODEL` | Model name | `gpt-4o` |
| `LLM_BASE_URL` / `OPENAI_BASE_URL` | API base URL | empty |
| `AGENT_WORKSPACE` | Workspace root | `~/agent_workspace` |
| `AGENT_SKILLS_DIR` | Extra Skills directory | built-in samples |
| `AGENT_HOST` | Listen host | `127.0.0.1` |
| `AGENT_PORT` | Listen port | `8899` |
| `AGENT_RECURSION_LIMIT` | Max reasoning steps | `60` |
| `AGENT_API_MAX_RETRIES` | Model connection retries | `3` |
| `AGENT_API_TIMEOUT_SECONDS` | Model request timeout seconds | `30` |
| `AGENT_CONTEXT_WINDOW_TOKENS` | Manual model context window | auto-detect |
| `AGENT_API_HOST_IPS` | Fallback DNS IP list for custom model gateways | empty |
| `DESKTOP_AGENT_AUTH_COOKIE_SECURE` | Send cookie over HTTPS only | `0` |

`AGENT_API_HOST_IPS` is only for DNS/network troubleshooting. The service does not include vendor IPs. Separate multiple IPs with commas.

### Image Input and Vision Models

The frontend supports pasting images directly into the composer. The backend converts images into OpenAI-compatible `image_url` messages.

Images are not written to SQLite history and are cleaned from LangGraph checkpoints after the turn, avoiding repeated image payloads in later text-only requests.

For MiMo models, if the current model name contains `mimo` but is not `mimo-v2.5` / `mimo-v2-omni`, image turns temporarily use `mimo-v2.5`. The saved default model is not changed.

## Usage

The composer supports:

- `Enter` for a newline
- `Cmd/Ctrl + Enter` to send
- Direct image paste
- Clicking **Stop** while a task is running

Example prompts:

```text
List workspace files
Write Python code to compute Fibonacci numbers and run it
Search today's AI news and summarize it
Open this link and summarize the page: https://example.com/article
Show the current changes in this repository
Commit the current changes with message: improve Skills support
Push the current branch
Revert the previous commit
Remember: I prefer English replies by default
```

### Git Tool Boundaries

Available Git tools:

```text
git_status
git_diff
git_log
git_show
git_add
git_commit
git_commit_all
git_push
git_revert
git_command
```

Safety policy:

- Use `git_add` / `git_commit` / `git_commit_all` only when the user explicitly asks to commit
- Use `git_push` only when the user explicitly asks to push
- Use `git_revert` only when the user explicitly asks to revert
- `git_push` only supports normal pushes, specifying remote/branch, or first-time upstream setup
- `git_revert` only supports a single revision and optional `--no-commit`
- High-risk operations such as `pull`, `reset`, `restore`, force push, range revert, and merge revert are not exposed

## Artifacts

If the Agent writes or appends a workspace file, the final reply automatically includes a downloadable files section.

Markdown files include:

- `Preview`: render Markdown in a modal
- `Download`: download the raw file

Other files only provide download links.

## Skills

AgentSmith loads `SKILL.md` from:

```text
agent_core/samples/
AGENT_SKILLS_DIR
project .opencode/skills/
project .claude/skills/
project .agents/skills/
```

It supports:

1. Simple Markdown sections: `Description`, `Trigger`, `Instructions`
2. YAML frontmatter compatible with oh-my-openagent / Superpowers style

The sidebar shows loaded skills in a fixed-height scrollable panel. Hovering a skill displays its description and triggers. When users ask which skills are available, the backend returns the real `SkillRegistry` contents instead of relying on model guesses.

## API

After startup, visit:

```text
http://127.0.0.1:8899/docs
```

Main endpoints:

| Endpoint | Method | Description |
| --- | --- | --- |
| `/` | GET | Desktop UI |
| `/login` | GET | Login page |
| `/auth/login` | POST | Login |
| `/auth/logout` | POST | Logout |
| `/auth/token-login` | GET | Short-lived token login |
| `/run` | POST | Non-streaming execution |
| `/run/stream` | POST | SSE streaming execution |
| `/sessions` | GET/POST | List or create sessions |
| `/sessions/{id}` | GET/DELETE | Get or delete a session |
| `/sessions/{id}/rename` | PUT | Rename a session |
| `/skills` | GET | List Skills |
| `/skills/reload` | POST | Reload Skills |
| `/memories` | GET/POST | Query or save long-term memory |
| `/memories/{key}` | DELETE | Delete memory |
| `/artifacts/download` | GET | Download workspace artifact |
| `/artifacts/preview` | GET | Preview Markdown artifact |
| `/settings` | GET/POST | Read or save configuration |
| `/usage` | GET | Today's usage |
| `/usage/session` | GET | Session usage |
| `/usage/history` | GET | Historical usage |
| `/users` | GET/POST | Manage users |
| `/users/{user_id}` | DELETE | Delete user |
| `/users/me` | GET | Current user |
| `/health` | GET | Health check |

All APIs except login, logout, token login, and health check require authentication.

## Remote Deployment

Example deployment to `/opt/desktop-agent` on port `8080`:

```bash
git clone https://gitee.com/mingwuda/desktop-agent.git /opt/desktop-agent
cd /opt/desktop-agent
python3 -m venv .venv
.venv/bin/python -m pip install --upgrade pip setuptools wheel
.venv/bin/python -m pip install -r requirements.txt
```

Example systemd unit:

```ini
[Unit]
Description=AgentSmith
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=/opt/desktop-agent
Environment=AGENT_HOST=0.0.0.0
Environment=AGENT_PORT=8080
Environment=AGENT_OPEN_BROWSER=0
Environment=DESKTOP_AGENT_AUTH_USER=admin
Environment=DESKTOP_AGENT_AUTH_PASSWORD=change-me
Environment=DESKTOP_AGENT_AUTH_SECRET=replace-with-random-secret
ExecStart=/opt/desktop-agent/.venv/bin/python /opt/desktop-agent/agent_core/main.py
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

Start and inspect:

```bash
systemctl daemon-reload
systemctl enable desktop-agent
systemctl restart desktop-agent
systemctl status desktop-agent
journalctl -u desktop-agent -f
```

For public deployments, place the service behind HTTPS reverse proxy, set a strong password, and use a fixed `DESKTOP_AGENT_AUTH_SECRET`.

## Windows Packaging

Build a distributable package:

```cmd
packaging\windows\build.cmd
```

Or:

```powershell
powershell -ExecutionPolicy Bypass -File .\packaging\windows\build.ps1
```

Output:

```text
dist\windows\DesktopAgent-Windows\
dist\windows\DesktopAgent-Windows.zip
```

After extraction, users can double-click `Start Desktop Agent.bat`. Configuration, sessions, and workspaces are stored in the user directory by default.

## Tech Stack

| Layer | Technology |
| --- | --- |
| Agent | LangGraph ReAct Agent |
| LLM interface | LangChain OpenAI, compatible with OpenAI-style APIs |
| Backend | FastAPI + Uvicorn |
| Frontend | Plain HTML/CSS/JS + marked.js |
| State | LangGraph MemorySaver + SQLite |
| Auth | HttpOnly Cookie + HMAC signature |
| Runtime | Python 3.9+ (3.10+ recommended) |
